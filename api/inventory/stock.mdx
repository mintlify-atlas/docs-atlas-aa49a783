---
title: Stock Queries
description: Query current stock levels across locations and variants
---

Stock queries provide real-time visibility into inventory levels. Each stock record represents a variant at a specific location with on-hand, reserved, and available quantities.

## List Stock Records

<CodeGroup>
```bash cURL
curl -X GET "https://api.sushigo.com/api/v1/stock?inventory_location_id=1&min_on_hand=1" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.sushigo.com/api/v1/stock?inventory_location_id=1', {
  headers: { 'Authorization': 'Bearer YOUR_TOKEN' }
});
```
</CodeGroup>

### Query Parameters

<ParamField query="inventory_location_id" type="integer">
  Filter by inventory location
</ParamField>

<ParamField query="item_variant_id" type="integer">
  Filter by item variant
</ParamField>

<ParamField query="min_on_hand" type="number">
  Filter by minimum on-hand quantity (e.g., `1` to show only items with stock)
</ParamField>

<ParamField query="per_page" type="integer" default="15">
  Items per page
</ParamField>

### Response

<ResponseField name="data" type="array">
  <Expandable title="Stock Record">
    <ResponseField name="id" type="integer">Stock record ID</ResponseField>
    <ResponseField name="inventory_location_id" type="integer">Location ID</ResponseField>
    <ResponseField name="item_variant_id" type="integer">Variant ID</ResponseField>
    <ResponseField name="on_hand" type="number">Total quantity on hand</ResponseField>
    <ResponseField name="reserved" type="number">Quantity reserved (e.g., for orders)</ResponseField>
    <ResponseField name="available" type="number">Available quantity (on_hand - reserved)</ResponseField>
    <ResponseField name="weighted_avg_cost" type="number">Weighted average unit cost</ResponseField>
    <ResponseField name="inventory_location" type="object">
      Location details (id, name, type, operating_unit)
    </ResponseField>
    <ResponseField name="item_variant" type="object">
      Variant details (id, code, name, item)
    </ResponseField>
  </Expandable>
</ResponseField>

## Stock by Location

Get a detailed stock summary for a specific inventory location, including all variants stored there.

<CodeGroup>
```bash cURL
curl -X GET "https://api.sushigo.com/api/v1/stock/by-location/1" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.sushigo.com/api/v1/stock/by-location/1', {
  headers: { 'Authorization': 'Bearer YOUR_TOKEN' }
});
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="integer" required>
  Inventory location ID
</ParamField>

### Response

<ResponseField name="data" type="object">
  <Expandable title="Location Stock Summary">
    <ResponseField name="inventory_location" type="object">
      <Expandable title="Location Info">
        <ResponseField name="id" type="integer">Location ID</ResponseField>
        <ResponseField name="name" type="string">Location name</ResponseField>
        <ResponseField name="type" type="string">Location type</ResponseField>
        <ResponseField name="operating_unit" type="string">Operating unit name</ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="summary" type="object">
      <Expandable title="Summary Totals">
        <ResponseField name="total_variants" type="integer">Number of variants in this location</ResponseField>
        <ResponseField name="total_on_hand" type="number">Total quantity on hand</ResponseField>
        <ResponseField name="total_reserved" type="number">Total reserved quantity</ResponseField>
        <ResponseField name="total_available" type="number">Total available quantity</ResponseField>
        <ResponseField name="total_inventory_value" type="number">Total inventory value (on_hand × weighted_avg_cost)</ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="items" type="array">
      <Expandable title="Item Details">
        <ResponseField name="item_variant_id" type="integer">Variant ID</ResponseField>
        <ResponseField name="item_variant_code" type="string">Variant code</ResponseField>
        <ResponseField name="item_variant_name" type="string">Variant name</ResponseField>
        <ResponseField name="item_name" type="string">Parent item name</ResponseField>

        <ResponseField name="weighted_avg_cost" type="number">Unit cost</ResponseField>

## Understanding Stock Fields

<Note>
**Stock Quantities:**
- `on_hand` - Physical quantity in the location
- `reserved` - Quantity allocated to orders or production
- `available` - Quantity available for new allocations (on_hand - reserved)

**Costing:**
- `weighted_avg_cost` - Weighted average cost calculated from stock movements
- Updated automatically when receiving inventory
</Note>

<Tip>
Use `min_on_hand=1` in the list endpoint to show only items with stock, or `min_on_hand=0.01` to include all variants that have ever had stock.
</Tip>
