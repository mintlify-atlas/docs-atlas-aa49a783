---
title: Item Variants
description: Manage item variants with different units of measure
---

Item variants represent specific presentations of an item (e.g., 1kg bag, 5kg box). Each variant has its own unit of measure, pricing, and stock tracking.

## List Item Variants

<CodeGroup>
```bash cURL
curl -X GET "https://api.sushigo.com/api/v1/item-variants?item_id=1&per_page=20" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.sushigo.com/api/v1/item-variants?item_id=1', {
  headers: { 'Authorization': 'Bearer YOUR_TOKEN' }
});
```
</CodeGroup>

## Create Item Variant

<CodeGroup>
```bash cURL
curl -X POST "https://api.sushigo.com/api/v1/item-variants" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "item_id": 1,
    "uom_id": 3,
    "code": "ARR-KG",
    "name": "Arroz Premium 1kg",
    "description": "Presentación de 1 kilogramo",
    "sale_price": 35.00,
    "min_stock": 10,
    "max_stock": 100,
    "track_lot": false,
    "is_active": true
  }'
```

```javascript JavaScript
const response = await fetch('https://api.sushigo.com/api/v1/item-variants', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    item_id: 1,
    uom_id: 3,
    code: 'ARR-KG',
    name: 'Arroz Premium 1kg',
    sale_price: 35.00
  })
});
```
</CodeGroup>

### Request Body

<ParamField body="item_id" type="integer" required>
  Parent item ID (must exist)
</ParamField>

<ParamField body="uom_id" type="integer" required>
  Base unit of measure ID
</ParamField>

<ParamField body="code" type="string" required>
  Unique variant code (auto-uppercased, max 100 chars)
</ParamField>

<ParamField body="name" type="string" required>
  Variant name (max 255 chars)
</ParamField>

<ParamField body="barcode" type="string">
  Product barcode (EAN, UPC, Code128, etc.) - cleaned and uppercased automatically
</ParamField>

<ParamField body="description" type="string">
  Variant description
</ParamField>

<ParamField body="track_lot" type="boolean" default="false">
  Track lot/batch numbers
</ParamField>

<ParamField body="track_serial" type="boolean" default="false">
  Track individual serial numbers
</ParamField>

<ParamField body="sale_price" type="number">
  Default sale price (≥ 0)
</ParamField>

<ParamField body="min_stock" type="number" default="0">
  Minimum stock level (≥ 0)
</ParamField>

<ParamField body="max_stock" type="number" default="0">
  Maximum stock level (≥ min_stock)
</ParamField>

<ParamField body="is_active" type="boolean" default="true">
  Active status
</ParamField>

### Response

Returns the created variant object with related item and UOM data. Status `201`.

### Errors

- `422` - Validation error (duplicate code, invalid item/UOM, max_stock < min_stock)
- `403` - Insufficient permissions

## Update Item Variant

<CodeGroup>
```bash cURL
curl -X PUT "https://api.sushigo.com/api/v1/item-variants/1" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "sale_price": 38.00,
    "min_stock": 15
  }'
```

```javascript JavaScript
const response = await fetch('https://api.sushigo.com/api/v1/item-variants/1', {
  method: 'PUT',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    sale_price: 38.00,
    min_stock: 15
  })
});
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="integer" required>
  Item variant ID
</ParamField>

### Request Body

All fields are optional. Only provided fields will be updated.

<ParamField body="code" type="string">
  Variant code (must be unique)
</ParamField>

<ParamField body="barcode" type="string">
  Product barcode
</ParamField>

<ParamField body="name" type="string">
  Variant name
</ParamField>

<ParamField body="description" type="string">
  Description
</ParamField>

<ParamField body="sale_price" type="number">
  Sale price
</ParamField>

<ParamField body="min_stock" type="number">
  Minimum stock
</ParamField>

<ParamField body="max_stock" type="number">
  Maximum stock
</ParamField>

<ParamField body="is_active" type="boolean">
  Active status
</ParamField>

<Note>
`item_id`, `uom_id`, `track_lot`, and `track_serial` cannot be changed after creation.
</Note>

### Response

Returns the updated variant object with status `200`.

### Errors

- `404` - Variant not found
- `422` - Validation error
- `403` - Insufficient permissions

## Delete Item Variant

<CodeGroup>
```bash cURL
curl -X DELETE "https://api.sushigo.com/api/v1/item-variants/1" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.sushigo.com/api/v1/item-variants/1', {
  method: 'DELETE',
  headers: { 'Authorization': 'Bearer YOUR_TOKEN' }
});
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="integer" required>
  Item variant ID
</ParamField>

### Response

```json
{
  "status": 200,
  "message": "Item variant deleted successfully"
}
```

### Errors

- `404` - Variant not found
- `409` - Cannot delete - variant has stock on hand. Clear inventory first.
- `403` - Insufficient permissions

<Warning>
Variants with stock on hand cannot be deleted. You must first consume or move all stock to zero before deletion.
</Warning>
