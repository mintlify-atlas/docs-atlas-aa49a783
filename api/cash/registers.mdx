## Overview

Cash registers represent physical or virtual points of sale. Each register belongs to a branch and can be of three types: ON_PREMISE (dine-in), DELIVERY, or EVENT (temporary operations).

---

## Create Cash Register

```http
POST /api/v1/cash-registers
```

### Request Body

<ParamField body="branch_id" type="integer" required>
  Branch ID where the register is located
</ParamField>

<ParamField body="operating_unit_id" type="integer">
  Operating unit ID (required for EVENT type registers)
</ParamField>

<ParamField body="code" type="string" required>
  Unique register code (max 50 characters)
</ParamField>

<ParamField body="name" type="string" required>
  Register name (max 255 characters)
</ParamField>

<ParamField body="type" type="string" required>
  Register type: `ON_PREMISE`, `DELIVERY`, or `EVENT`
</ParamField>

<ParamField body="is_active" type="boolean" default="true">
  Active status
</ParamField>

<ParamField body="meta" type="object">
  Additional metadata
  
  <Expandable title="properties">
    <ResponseField name="location" type="string">
      Physical location description
    </ResponseField>
  </Expandable>
</ParamField>

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      Register ID
    </ResponseField>
    
    <ResponseField name="branch_id" type="integer">
      Branch ID
    </ResponseField>
    
    <ResponseField name="operating_unit_id" type="integer">
      Operating unit ID (for events)
    </ResponseField>
    
    <ResponseField name="code" type="string">
      Register code
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Register name
    </ResponseField>
    
    <ResponseField name="type" type="string">
      Register type
    </ResponseField>
    
    <ResponseField name="is_active" type="boolean">
      Active status
    </ResponseField>
    
    <ResponseField name="meta" type="object">
      Additional metadata
    </ResponseField>
    
    <ResponseField name="branch" type="object">
      Branch information
    </ResponseField>
    
    <ResponseField name="operating_unit" type="object">
      Operating unit information (if applicable)
    </ResponseField>
  </Expandable>
</ResponseField>

## Delete Cash Register

<Warning>
Cannot delete a register with existing sessions. Set `is_active` to `false` instead.
</Warning>

```http
DELETE /api/v1/cash-registers/{id}
```

### Path Parameters

<ParamField path="id" type="integer" required>
  Cash register ID
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<RequestExample>
```bash cURL
curl -X DELETE https://api.sushigo.local/api/v1/cash-registers/1 \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</RequestExample>

<ResponseExample>
```json Success Response
{
  "message": "Cash register deleted successfully"
}
```

```json 422 Cannot Delete
{
  "message": "Cannot delete cash register with existing sessions"
}
```
</ResponseExample>

## Register Types

<Tabs>
  <Tab title="ON_PREMISE">
    Used for dine-in sales at the restaurant location. This is the most common type for traditional POS operations.
  </Tab>
  
  <Tab title="DELIVERY">
    Dedicated register for delivery orders. Separates delivery cash flow from on-premise operations.
  </Tab>
  
  <Tab title="EVENT">
    Temporary register for catering events or pop-up locations. Linked to an operating unit with specific dates.
  </Tab>
</Tabs>
