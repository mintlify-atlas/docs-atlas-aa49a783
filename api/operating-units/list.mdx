## List Operating Units

<api method="GET" endpoint="/api/v1/operating-units" />

Retrieves a paginated list of operating units with optional filtering by search term, type, active status, and branch.

### Query Parameters

<ParamField query="per_page" type="integer" default="15">
  Number of items per page (1-100)
</ParamField>

<ParamField query="search" type="string">
  Search by operating unit name (partial match, case-insensitive)
</ParamField>

<ParamField query="type" type="string">
  Filter by operating unit type. Valid values:
  - `BRANCH_MAIN` - Main branch operating unit
  - `BRANCH_BUFFER` - Branch buffer storage
  - `BRANCH_RETURN` - Branch return processing
  - `EVENT_TEMP` - Temporary event unit
</ParamField>

<ParamField query="is_active" type="boolean">
  Filter by active status (`true` or `false`)
</ParamField>

<ParamField query="branch_id" type="integer">
  Filter by branch ID
</ParamField>

### Response

<ResponseField name="data" type="array">
  Array of operating unit objects

  <ResponseField name="id" type="integer">
    Operating unit ID
  </ResponseField>

  <ResponseField name="branch_id" type="integer">
    Associated branch ID
  </ResponseField>

  <ResponseField name="name" type="string">
    Operating unit name
  </ResponseField>

  <ResponseField name="type" type="string">
    Operating unit type (BRANCH_MAIN, BRANCH_BUFFER, BRANCH_RETURN, EVENT_TEMP)
  </ResponseField>

  <ResponseField name="start_date" type="string">
    Start date (YYYY-MM-DD format)
  </ResponseField>

  <ResponseField name="end_date" type="string" nullable>
    End date (YYYY-MM-DD format, null for permanent units)
  </ResponseField>

  <ResponseField name="is_active" type="boolean">
    Whether the operating unit is currently active
  </ResponseField>

  <ResponseField name="meta" type="object" nullable>
    Additional metadata (JSON object)
  </ResponseField>

  <ResponseField name="branch" type="object">
    Associated branch details
    
    <ResponseField name="id" type="integer">
      Branch ID
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Branch name
    </ResponseField>
  </ResponseField>

  <ResponseField name="created_at" type="string">
    Creation timestamp (ISO 8601 UTC)
  </ResponseField>

  <ResponseField name="updated_at" type="string">
    Last update timestamp (ISO 8601 UTC)
  </ResponseField>
</ResponseField>

<ResponseField name="current_page" type="integer">
  Current page number
</ResponseField>

<ResponseField name="per_page" type="integer">
  Items per page
</ResponseField>

<ResponseField name="total" type="integer">
  Total number of operating units
</ResponseField>

<ResponseField name="last_page" type="integer">
  Last page number
</ResponseField>

### Example Request

<Tabs>
  <Tab title="cURL">
    ```bash
    curl -X GET "https://api.sushigo.local/api/v1/operating-units?per_page=10&type=BRANCH_MAIN&is_active=true" \
      -H "Accept: application/json"
    ```
  </Tab>
  
  <Tab title="JavaScript">
    ```javascript
    const response = await fetch('https://api.sushigo.local/api/v1/operating-units?per_page=10&type=BRANCH_MAIN&is_active=true', {
      method: 'GET',
      headers: {
        'Accept': 'application/json'
      }
    });
    
    const data = await response.json();
    ```

</Tabs>

## Show Operating Unit

<api method="GET" endpoint="/api/v1/operating-units/{id}" />

Retrieves a single operating unit by ID, including associated branch and inventory locations.

### Path Parameters

<ParamField path="id" type="integer" required>
  Operating unit ID
</ParamField>

### Response

<ResponseField name="data" type="object">
  Operating unit object

  <ResponseField name="id" type="integer">
    Operating unit ID
  </ResponseField>

  <ResponseField name="branch_id" type="integer">
    Associated branch ID
  </ResponseField>

  <ResponseField name="name" type="string">
    Operating unit name
  </ResponseField>

  <ResponseField name="type" type="string">
    Operating unit type (BRANCH_MAIN, BRANCH_BUFFER, BRANCH_RETURN, EVENT_TEMP)
  </ResponseField>

  <ResponseField name="start_date" type="string">
    Start date (YYYY-MM-DD format)
  </ResponseField>

  <ResponseField name="end_date" type="string" nullable>
    End date (YYYY-MM-DD format, null for permanent units)
  </ResponseField>

  <ResponseField name="is_active" type="boolean">
    Whether the operating unit is currently active
  </ResponseField>

  <ResponseField name="meta" type="object" nullable>
    Additional metadata (JSON object)
  </ResponseField>

  <ResponseField name="branch" type="object">
    Associated branch details
    
    <ResponseField name="id" type="integer">
      Branch ID
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Branch name
    </ResponseField>
  </ResponseField>

  <ResponseField name="inventory_locations" type="array">
    Array of inventory location objects associated with this operating unit
    
    <ResponseField name="id" type="integer">
      Location ID
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Location name
    </ResponseField>
    
    <ResponseField name="is_primary" type="boolean">
      Whether this is the primary location for the operating unit
    </ResponseField>
  </ResponseField>

  <ResponseField name="created_at" type="string">
    Creation timestamp (ISO 8601 UTC)
  </ResponseField>

  <ResponseField name="updated_at" type="string">
    Last update timestamp (ISO 8601 UTC)
  </ResponseField>
</ResponseField>

### Example Request
