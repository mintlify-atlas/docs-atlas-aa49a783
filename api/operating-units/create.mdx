## Create Operating Unit

<api method="POST" endpoint="/api/v1/operating-units" auth="bearer" />

Creates a new operating unit within a branch. Operating units represent operational contexts such as main branch inventory, buffer storage, return processing areas, or temporary event locations.

### Request Body

<ParamField body="branch_id" type="integer" required>
  ID of the branch this operating unit belongs to
</ParamField>

<ParamField body="name" type="string" required>
  Operating unit name (max 255 characters)
</ParamField>

<ParamField body="type" type="string" required>
  Operating unit type. Valid values:
  - `BRANCH_MAIN` - Main branch operating unit
  - `BRANCH_BUFFER` - Branch buffer storage
  - `BRANCH_RETURN` - Branch return processing
  - `EVENT_TEMP` - Temporary event unit
</ParamField>

<ParamField body="start_date" type="string">
  Start date in YYYY-MM-DD format
</ParamField>

<ParamField body="end_date" type="string">
  End date in YYYY-MM-DD format (must be after or equal to start_date). Used primarily for EVENT_TEMP types.
</ParamField>

<ParamField body="is_active" type="boolean" default="true">
  Whether the operating unit is active
</ParamField>

<ParamField body="meta" type="object">
  Additional metadata as a JSON object
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="data" type="object">
  Created operating unit object

  <ResponseField name="id" type="integer">
    Operating unit ID
  </ResponseField>

  <ResponseField name="branch_id" type="integer">
    Associated branch ID
  </ResponseField>

  <ResponseField name="name" type="string">
    Operating unit name
  </ResponseField>

  <ResponseField name="type" type="string">
    Operating unit type
  </ResponseField>

  <ResponseField name="start_date" type="string">
    Start date (YYYY-MM-DD format)
  </ResponseField>

  <ResponseField name="end_date" type="string" nullable>
    End date (YYYY-MM-DD format)
  </ResponseField>

  <ResponseField name="is_active" type="boolean">
    Active status
  </ResponseField>

  <ResponseField name="meta" type="object" nullable>
    Additional metadata
  </ResponseField>

  <ResponseField name="branch" type="object">
    Associated branch details
    
    <ResponseField name="id" type="integer">
      Branch ID
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Branch name
    </ResponseField>
  </ResponseField>

  <ResponseField name="created_at" type="string">
    Creation timestamp (ISO 8601 UTC)
  </ResponseField>

  <ResponseField name="updated_at" type="string">
    Last update timestamp (ISO 8601 UTC)
  </ResponseField>
</ResponseField>

      -H "Authorization: Bearer YOUR_TOKEN" \
      -H "Content-Type: application/json" \

      -d '{

## Update Operating Unit

<api method="PUT" endpoint="/api/v1/operating-units/{id}" auth="bearer" />

Updates an existing operating unit. All fields are optional; only provided fields will be updated.

### Path Parameters

<ParamField path="id" type="integer" required>
  Operating unit ID
</ParamField>

### Request Body

<ParamField body="branch_id" type="integer">
  ID of the branch this operating unit belongs to
</ParamField>

<ParamField body="name" type="string">
  Operating unit name (max 255 characters)
</ParamField>

<ParamField body="type" type="string">
  Operating unit type (BRANCH_MAIN, BRANCH_BUFFER, BRANCH_RETURN, EVENT_TEMP)
</ParamField>

<ParamField body="start_date" type="string">
  Start date in YYYY-MM-DD format
</ParamField>

<ParamField body="end_date" type="string">
  End date in YYYY-MM-DD format (must be after or equal to start_date)
</ParamField>

<ParamField body="is_active" type="boolean">
  Whether the operating unit is active
</ParamField>

<ParamField body="meta" type="object">
  Additional metadata as a JSON object
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="data" type="object">
  Updated operating unit object (same structure as create response)
</ResponseField>

## Delete Operating Unit

<api method="DELETE" endpoint="/api/v1/operating-units/{id}" auth="bearer" />

Soft deletes an operating unit. The unit will be marked as deleted but retained in the database for audit purposes.

### Path Parameters

<ParamField path="id" type="integer" required>
  Operating unit ID
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message
</ResponseField>
