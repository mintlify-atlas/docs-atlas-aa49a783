## List Users in Operating Unit

<api method="GET" endpoint="/api/v1/operating-units/{id}/users" />

Retrieves a paginated list of users assigned to a specific operating unit, including their roles.

### Path Parameters

<ParamField path="id" type="integer" required>
  Operating unit ID
</ParamField>

### Query Parameters

<ParamField query="per_page" type="integer" default="15">
  Number of items per page (1-100)
</ParamField>

### Response

<ResponseField name="data" type="array">
  Array of user objects

  <ResponseField name="id" type="integer">
    User ID
  </ResponseField>

  <ResponseField name="name" type="string">
    User's full name
  </ResponseField>

  <ResponseField name="email" type="string">
    User's email address
  </ResponseField>

  <ResponseField name="roles" type="array">
    Array of role objects assigned to the user
    
    <ResponseField name="id" type="integer">
      Role ID
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Role name
    </ResponseField>
  </ResponseField>

  <ResponseField name="pivot" type="object">
    Pivot table data for the user-operating unit relationship
    
    <ResponseField name="operating_unit_id" type="integer">
      Operating unit ID
    </ResponseField>
    
    <ResponseField name="user_id" type="integer">
      User ID
    </ResponseField>
    
    <ResponseField name="assignment_role" type="string" nullable>
      Role specific to this assignment
    </ResponseField>
    
    <ResponseField name="is_active" type="boolean">
      Whether the assignment is active
    </ResponseField>
    
    <ResponseField name="meta" type="object" nullable>
      Additional assignment metadata
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      Assignment creation timestamp
    </ResponseField>
    
    <ResponseField name="updated_at" type="string">
      Assignment last update timestamp
    </ResponseField>
  </ResponseField>
</ResponseField>

<ResponseField name="current_page" type="integer">
  Current page number
</ResponseField>

<ResponseField name="per_page" type="integer">
  Items per page
</ResponseField>

<ResponseField name="total" type="integer">
  Total number of users
</ResponseField>

<ResponseField name="last_page" type="integer">
  Last page number
</ResponseField>

### Example Request

<Tabs>
  <Tab title="cURL">
    ```bash
    curl -X GET "https://api.sushigo.local/api/v1/operating-units/1/users?per_page=10" \
      -H "Accept: application/json"
    ```
  </Tab>
  
  <Tab title="JavaScript">
    ```javascript
    const response = await fetch('https://api.sushigo.local/api/v1/operating-units/1/users?per_page=10', {
      method: 'GET',
      headers: {
        'Accept': 'application/json'
      }
    });
    
    const data = await response.json();
    ```
  </Tab>
</Tabs>

## Add User to Operating Unit

<api method="POST" endpoint="/api/v1/operating-units/{id}/users" auth="bearer" />

Assigns a user to an operating unit. Users must be assigned to an operating unit to perform inventory operations within that unit.

### Path Parameters

<ParamField path="id" type="integer" required>
  Operating unit ID
</ParamField>

### Request Body

<ParamField body="user_id" type="integer" required>
  ID of the user to assign to the operating unit
</ParamField>

### Response

<ResponseField name="data" type="object">
  Assignment confirmation

  <ResponseField name="message" type="string">
    Success message
  </ResponseField>

  <ResponseField name="operating_unit" type="object">
    Operating unit summary
    
    <ResponseField name="id" type="integer">
      Operating unit ID
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Operating unit name
    </ResponseField>
    
    <ResponseField name="type" type="string">
      Operating unit type
    </ResponseField>
  </ResponseField>

  <ResponseField name="user" type="object">
    User summary
    
    <ResponseField name="id" type="integer">
      User ID
    </ResponseField>
    
    <ResponseField name="name" type="string">
      User name
    </ResponseField>
    
    <ResponseField name="email" type="string">
      User email
    </ResponseField>
  </ResponseField>
</ResponseField>

<ResponseField name="status" type="integer">
  HTTP status code (200)
</ResponseField>

<ResponseField name="meta" type="object">
  Response metadata
</ResponseField>

### Example Request

<Tabs>
  <Tab title="cURL">
    ```bash
    curl -X POST "https://api.sushigo.local/api/v1/operating-units/1/users" \
      -H "Authorization: Bearer YOUR_TOKEN" \
      -H "Content-Type: application/json" \
      -H "Accept: application/json" \
      -d '{
        "user_id": 3
      }'
    ```
  </Tab>
  
  <Tab title="JavaScript">
    ```javascript
    const response = await fetch('https://api.sushigo.local/api/v1/operating-units/1/users', {
      method: 'POST',
      headers: {
        'Authorization': 'Bearer YOUR_TOKEN',
        'Content-Type': 'application/json',
        'Accept': 'application/json'
      },
      body: JSON.stringify({
        user_id: 3
      })
    });
    
    const data = await response.json();
    ```
  </Tab>
</Tabs>

### Example Response

```json
{
  "data": {
    "message": "User added to operating unit successfully",
    "operating_unit": {
      "id": 1,
      "name": "Main Kitchen",
      "type": "BRANCH_MAIN"
    },
    "user": {
      "id": 3,
      "name": "John Smith",
      "email": "john.smith@sushigo.com"
    }
  },
  "status": 200,
  "meta": {}
}
```

### Error Responses

<ResponseField name="404" type="object">
  Operating unit or user not found
  
  ```json
  {
    "message": "Not found"
  }
  ```
</ResponseField>

<ResponseField name="409" type="object">
  User already assigned to operating unit
  
  ```json
  {
    "status": 409,
    "message": "User is already assigned to this operating unit",
    "errors": []
  }
  ```
</ResponseField>

<ResponseField name="422" type="object">
  Validation error
  
  ```json
  {
    "message": "The given data was invalid.",
    "errors": {
      "user_id": ["The user id field is required."]
    }
  }
  ```
</ResponseField>

## Remove User from Operating Unit

<api method="DELETE" endpoint="/api/v1/operating-units/{id}/users/{userId}" auth="bearer" />

Removes a user assignment from an operating unit. This prevents the user from performing operations in this specific operating unit.

### Path Parameters

<ParamField path="id" type="integer" required>
  Operating unit ID
</ParamField>

<ParamField path="userId" type="integer" required>
  User ID to remove from the operating unit
</ParamField>

### Response

<ResponseField name="data" type="object">
  Removal confirmation

  <ResponseField name="message" type="string">
    Success message
  </ResponseField>
</ResponseField>

<ResponseField name="status" type="integer">
  HTTP status code (200)
</ResponseField>

<ResponseField name="meta" type="object">
  Response metadata
</ResponseField>

### Example Request

<Tabs>
  <Tab title="cURL">
    ```bash
    curl -X DELETE "https://api.sushigo.local/api/v1/operating-units/1/users/3" \
      -H "Authorization: Bearer YOUR_TOKEN" \
      -H "Accept: application/json"
    ```
  </Tab>
  
  <Tab title="JavaScript">
    ```javascript
    const response = await fetch('https://api.sushigo.local/api/v1/operating-units/1/users/3', {
      method: 'DELETE',
      headers: {
        'Authorization': 'Bearer YOUR_TOKEN',
        'Accept': 'application/json'
      }
    });
    
    const data = await response.json();
    ```
  </Tab>
</Tabs>

### Example Response

```json
{
  "data": {
    "message": "User removed from operating unit successfully"
  },
  "status": 200,
  "meta": {}
}
```

### Error Responses

<ResponseField name="404" type="object">
  Operating unit not found or user not assigned
  
  ```json
  {
    "status": 404,
    "message": "User is not assigned to this operating unit",
    "errors": []
  }
  ```
</ResponseField>
